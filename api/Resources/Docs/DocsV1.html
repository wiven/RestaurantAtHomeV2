User actions

/login

allows to authenticate the user.

example: http://.../login/email@hotmail.com/a361f49bee3c15f9691c92506d14361e/0

Type

GET

Parameters

  • email
  • password
  • socialLogin: indicates the user authenticated with a social media. Then a password isn't required

Return


                {
                "hash": "8aab72bb43951f7a3ebbb009682a229e42b8d6a9",
                "email": "thdepauw@hotmail.com",
                "name": "Thomas",
                "surname": "De Pauw",
                "type": "Client"
                }

/user

Create a new user.

example: http://.../user

Type

POST

Parameters

This has to be send in the body:


                {
                "email":"email@hotmail.com",
                "name":"Inessa",
                "surname":"Verleyen",
                "phoneNo":"0123456789",
                "type":"Client",
                "socialLogin": "0",
                "password":"a361f49bee3c15f9691c92506d14361e"
                }

Extra info

  • socialLogin: indicates the user authenticated with a social media. Then a password isn't required
  • password: Send as MD5
  • Type:
    • Client
    • Resto

Return


                {
                "hash": "ac3478d69a3c81fa62e60f5c3696165a4e5e6ac4",
                "email": "inessa_verleyen@hotmail.com",
                "name": "Inessa",
                "surname": "Verleyen",
                "phoneNo": "0123456789",
                "type": "Client",
                "socialLogin": "0"
                }
